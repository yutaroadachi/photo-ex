サービス名    ：PHOTO EX
サービス概要  ：PHOTO EX（Photo Exhibition）はインターネット上の写真展です。
                お気に入りの写真を展示したり、他の人が展示している写真を鑑賞したりすることが可能です。
                気に入ったユーザーをフォローしたり、気に入った写真にいいねやコメントをしたりすることも可能です。
                あなたの写真展を開いてみませんか？
利用方法      ：サイトURLにアクセスの上、ヘッダーメニューからサインアップ、またはログインをクリックし、
                遷移先画面にて必要情報を入力してサインアップ、またはログインを行なってください。
                遷移祭画面にて「Facebookで認証」リンクをクリックすると、Facebookによるサインアップ、またはログインも可能です。
サイトURL     ：https://photo-ex.herokuapp.com/
テストユーザー：Eメール：example@railstutorial.org、パスワード：foobar

【苦労した点】
・いいね機能の実装で一時期エラーがいつまで経っても原因が分からず、解決に苦労した。
　原因はMVCと変数の受け渡しの流れを意識せずにコードを書いており、
　ビューで定義した変数をコントローラーで定義しておらず渡せていないことだった。
　MVCとWebの仕組みについて勉強を進め、HTTPリクエストが送られてからレスポンスが返されるまでの
　MVCの役割と情報の受け渡しに関して一気通貫した設計を意識し、実装するようにしたい。
・コメント機能の実装でエラーが発生したが、原因がモデルのバリデーションのカラムのタイポだったこともあり、
　エラーメッセージの内容から原因を推測することも、ググって適当な情報を見つけ出すことも難しく、解決に苦労した。
　コメント機能で修正を加えたファイルを洗い出し、エラーメッセージの内容から最低限の優先順位をつけて
　ファイルを確認していくなど、冷静に状況を整理すればもう少し効率的にエラーの解決が行えたと思うので、次に活かしたい。
・Deviseを用いて認証周りの機能を実装したため、チュートリアルとの差異が発生し、
　エラーの原因となってその解決に苦労した。

【学んだ点】
・機能を実装していく時の基本を学んだ。
　終盤の実装では、ある程度ビューで表示したい情報についてどのモデルからどういったデータを取ってきて、
　どのコントローラーにどういった処理を書いてビューに渡してやればいいかのイメージがつくようになった。
・Gemの仕様や実装方法の把握の仕方を学んだ。
　Qiitaやブログなどの日本語の情報だけでなく、GitHubにあるREADMEやWikiに書いてある仕様や実装方法を
　確認しながら実装できるようになった。
　Deviseの実装により、テストではDeviseが用意しているヘルパーメソッドを使用してログインしなければならないが、
　テストでヘルパーモジュールをインクルードしておらず、エラーが出てしまった時もその仕様をWikiを読んで把握していたため、
　迅速にエラーの解決ができた。
・gitの使い方の基本を学んだ。
　前のコミットに戻ったり、ブランチを消して失敗した作業をなかったことにしたりと状況に応じてコードの状態を
　適切に行ったり来たりすることができた。
　細かくブランチを切っていく重要性も学んだ。
・簡単ではあるがbyebugを使用したデバッグの仕方を学んだ。
　特にオブジェクトがnilになってしまうことでNoMethodErrorが発生してしまうケースは開発中に多く出くわしたが、
　byebugを使用することでどのカラムのデータがnilになってしまっているのかがわかり、迅速にエラーの解決ができた。
　今後はもう少し高度なデバッグにも挑戦してみたい。
　
【自慢したい点】
・仕様に挙げられている機能のほとんどを実装できた点。（モーダルウィンドウと通知機能以外）
・DeviseとRansackを用いずに認証と検索の機能を実装することは可能だったが、これらのGemの使用に挑戦し、
　機能の実装を成し遂げた点。
・いいね機能のAjax化を行なっている点。
・エラーメッセージなどメッセージの日本語化を行なっている点。
・なんとなく開発するのではなく、「インターネット上の写真展（写真展は英語で"Photo Exhibition"）」というコンセプトを
　最初に決めた上で開発に臨んだ点。
